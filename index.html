2. ContactService - Algorithm and Data Structure

This project was based on an assignment from CS320: Software Testing, Automation, and Quality Assurance (April 2024). It was a simple Java program that allowed users to add, delete, and update contacts. By choosing this artifact, I aimed to showcase my ability to select the right data structure and demonstrate how I write more efficient and well-structured methods.

Looking back at my code, I identified several areas for improvement, including choosing the right algorithm and data structure, improving encapsulation, and enhancing validation and error handling. I decided to replace the existing data structure with a HashMap to achieve O(1) time complexity, instead of using an ArrayList.

Code review video:

Enhancements:

  • Using a HashMap: I replaced the current ArrayList with a HashMap to improve the efficiency of data retrieval, addition, and deletion. The HashMap uses the contact ID as the key, allowing for constant-time (O(1)) operations for searching, updating, and deleting contacts (on ContactService.java).
  • Input Validation and Error Handling: I implemented input validation and error handling mechanisms to prevent invalid input and ensure errors were properly managed, preventing the application from crashing (on Contacts.java).
  • Flexibility and Scalability: Previously, I had four separate methods for updating a contact. I consolidated them into a single method, making the class more adaptable to changes without creating extra methods (on ContactService.java).

3. AnimalShelter.py - Database

This project is based on an assignment from CS 340: Advanced Programming Concepts in June 2024. It is a simple Python implementation of CRUD (Create, Read, Update, Delete) operations using MongoDB as the database. The project focuses on interacting with the AAC database, particularly the animals collection, to manage a list of animals.

The application was developed in Python. Looking back at my code, I implemented CRUD operations with practical error handling, added an aggregation pipeline to manage data more efficiently, and created indexes for each animal to improve search performance.

Key Aspects of This CRUD Implementation

  • Database Connectivity: How the program connects to the database.
  • Efficiency: The use of an aggregation pipeline and indexes to optimize queries.
  • Error Handling: The program provides clear error messages when issues occur.

Even if an error occurs, the program will output a detailed explanation of what happened.
